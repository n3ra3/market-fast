# market-fastbuy

Быстрый, лёгкий и стабильный бот для Market.CSGO. Единственная задача —
**мгновенно купить предмет из WatchList, если его цена ниже твоего лимита.**
Бот не считает прибыль, не анализирует Steam, не сравнивает цены и комиссии.
Вся «умная» часть — это лимит, который ставишь ты.

```
Centrifugo (WebSocket) ──► scanner ──► asyncio.Queue ──► buyer ──► /api/v2/buy
        │                     │                            │
   public:items          name_id ∈ WatchList?         rate limiter
   :730:<cur>            цена < лимита?                (приоритет покупке)
```

## Что внутри

| Файл | Роль |
|------|------|
| `config.py` | всё окружение и константы, плюс загрузка `.env` |
| `rate_limiter.py` | строгий лимит ≤ 4 req/s, без всплесков, приоритет покупке |
| `state.py` | состояние в RAM: WatchList, лимиты, статистика |
| `market_client.py` | один общий `aiohttp` сеанс (keep-alive), ws-токен, `/buy`, прогрев |
| `names.py` | один раз качает `names.json`, держит `name_id → hash_name` в памяти |
| `centrifugo_client.py` | корректный протокол Centrifugo (офиц. клиент), авто-reconnect и обновление 10-мин токена |
| `scanner.py` | из пуша достать name_id/цену/offer_id, отфильтровать, положить в очередь |
| `buyer.py` | читает очередь, мгновенный `/buy`, обработка HTTP 500, анти-дубль |
| `telegram.py` | информирование + установка лимитов (не на пути покупки) |
| `main.py` | сборка всего, `/health`, корректный старт/остановка |
| `watchlist.json` | постоянный список предметов (лимитов тут нет) |

## Модель безопасности (важно)

- **После любого старта/рестарта:** WatchList загружен, бот слушает WebSocket,
  но **лимитов нет → покупка выключена → чистый SCAN.** Ничего не купится,
  пока ты вручную не задашь лимит в Telegram.
- **Лимиты живут только в памяти** и никогда не пишутся на диск. Перезапуск
  Render их стирает — это защита от случайной покупки по старому лимиту.
- **Дороже лимита купить нельзя:** в `/buy` всегда уходит `price = твой лимит`,
  а Market покупает «по этой цене или ниже». Даже при ошибке парсинга цены
  переплаты не будет (серверная гарантия).
- **HTTP 500 не приводит к слепому ретраю** — результат помечается как
  «неизвестен», шлётся уведомление; авто-повтор покупки не делается.

## Первый запуск — режим DISCOVERY (один раз)

Точную схему пуша канала `public:items:730:<cur>` нужно один раз увидеть вживую.

1. Поставь в окружении `DISCOVERY_MODE=1`.
2. Запусти бота. Он подключится, подпишется и залогирует первые ~20 сырых
   пушей строками `RAW PUSH #N :: {...}`, **ничего не покупая.**
3. Пришли мне 2–3 такие строки. По ним финализируем парсер (откуда брать
   name_id, цену и offer_id). Если поля отличаются — правятся через env,
   **без изменения кода:**
   ```
   WS_NAME_ID_KEYS=name_id,nameId,i,n
   WS_PRICE_KEYS=price,value,p,c
   WS_OFFER_ID_KEYS=id,offer_id,offerId,o
   WS_PRICE_FORMAT=value   # value (0.142) или units (142)
   ```
4. Выстави `DISCOVERY_MODE=0` — бот готов покупать (после установки лимитов).

## Установка и запуск (локально)

```bash
cp .env.example .env          # впиши MARKET_API_KEY и Telegram
pip install -r requirements.txt
python main.py
```

Требуется Python 3.12+.

## Docker

```bash
docker build -t market-fastbuy .
docker run --env-file .env -p 8000:8000 market-fastbuy
```

## Render

- Тип сервиса: **Web Service** (нужен открытый порт под health-check).
- Регион: **Frankfurt** (ближе всего к инфраструктуре Market.CSGO → меньше
  задержка покупки).
- Render сам передаёт `PORT`; бот поднимает на нём `GET /health`.
- Переменные окружения — из `.env.example` (как минимум `MARKET_API_KEY`,
  `MARKET_TELEGRAM_TOKEN`, `MARKET_TELEGRAM_CHAT_ID`).
- Запускается из Docker без изменений.

### UptimeRobot

Чтобы Render не засыпал — пингуй health-эндпоинт:

```
GET https://<твой-сервис>.onrender.com/health   ->   {"status":"ok"}
```

## Telegram

- `/menu` — статус + меню: режим (SCAN/ARMED), WS, WatchList с лимитами.
- **⚙ Set · <предмет>** — бот спросит цену в USD; ответом ставишь лимит
  (предмет становится ARMED). Бот подтвердит точный `hash_name`, который
  зафиксировал — проверь, что это нужный предмет.
- **✖ Clear · <предмет>** — снять лимит.
- **🛑 DISARM ALL** — снять все лимиты, вернуться в SCAN.
- Уведомления: покупка (с задержкой WS→buy в мс), «результат неизвестен»
  (HTTP 500), ошибки.

## Зависимости

`aiohttp`, `centrifuge-python`, `loguru`. Только `asyncio`. Без `requests`,
потоков и multiprocessing.
